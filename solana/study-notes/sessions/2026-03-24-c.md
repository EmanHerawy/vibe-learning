## One-Paragraph Summary

A Solana transaction is a signed bundle of instructions, each containing three things: program_id, accounts (as AccountMeta with is_signer/is_writable flags), and serialized instruction data. All accounts across all instructions are deduplicated into one flat list at the message level — saving space within the 1232-byte limit and enabling the runtime to detect scheduling conflicts upfront. All instructions in a transaction are atomic — if any fails, all are rolled back. Transactions include a recent_blockhash for replay protection that expires in ~150 blocks (~1 minute). The Compute Budget Program lets you set priority fees (Set Compute Unit Price) and compute limits (Set Compute Unit Limit) — Solana's equivalent of gasPrice and gasLimit.
