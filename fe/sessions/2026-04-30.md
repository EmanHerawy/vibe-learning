## Theory

### `call.call<M>()` — definition

> Source: `effects.fe:368` — `pub trait Call`

```fe
fn call<M>(
    mut self,
    addr: own Address,
    gas: u256,
    value: u256,
    message: own M,        // ← ALWAYS required — cannot be omitted
) -> M::Return
where M: MsgVariant<Sol> + Encode<Sol>, M::Return: Decode<Sol> + AbiSize
```

**What it does internally** (`effects.fe:653–663`):

```
1. encode_calldata(M::SELECTOR, message)  → (args_offset, args_len) in scratch memory
2. raw_call(addr, gas, value, args_offset, args_len)
         ↓ if false → bubble_last_revert()   ← auto-bubbles callee revert
3. decode_returndata(last_returndata())    → M::Return
```

### Why empty calldata / skip `message` is impossible in `call.call()`

> Source: `effects.fe:368–369`, `fe-vs-solidity.md` (confirmed in practice)

| Solidity | Fe |
|---|---|
| `addr.call{value: x}("")` — empty calldata OK | `call.call(...)` — `message` required by type system |
| Triggers `receive()` on target | No `receive()` or `fallback()` in Fe |
| `""` = zero-length bytes | No `Unit`/`()` shorthand in `MsgVariant<Sol>` |

Trait bound `where M: MsgVariant<Sol>` means `M` must be a declared `msg` variant with `#[selector = sol("...")]`. There is no escape hatch.

```fe
call.call(addr: to, gas: 2300, value: amount)          // ❌ compiler error — message required
call.call(addr: to, gas: 2300, value: amount, message: "")  // ❌ not a MsgVariant
```

### `call.raw_call()` — the real bare-ETH-send ✅ NEW CONFIRMED

> Source: `effects.fe:349` (default impl on `pub trait Call`), confirmed usage in `recv_fallback.fe:34–37`

```fe
fn raw_call(
    mut self,
    addr: own Address,
    gas: u256,
    value: u256,
    args_offset: u256,   // memory pointer to calldata
    args_len: u256,      // 0 = empty calldata
) -> bool               // returns bool — does NOT auto-bubble revert
```

**Bare ETH send with empty calldata:**

```fe
let ok = call.raw_call(
    addr: to,
    gas: 100000,
    value: balance,
    args_offset: 0,   // ignored when args_len == 0
    args_len: 0,      // ← empty calldata = Solidity's addr.call{value: x}("")
)
// must check ok manually — no auto-revert
```

```
EVM CALL opcode
  ├── calldata = empty (len 0)
  ├── EOA target → no code → succeeds immediately
  └── ETH transfers ✅
```

**Key: `raw_call` has a default impl on the `Call` trait** — any `call: mut Call` effect handle in a production contract gets it automatically. No `evm: mut Evm` needed.

### `raw_call` return value — `bool` and why you must check it

> Source: `effects.fe:349–366`, confirmed via 7/7 Hardhat tests

```fe
fn raw_call(...) -> bool {
    ops::call(...) != 0   // EVM CALL returns 1 on success, 0 on failure → bool
}
```

| Return | Meaning |
|---|---|
| `true` | Call succeeded — ETH transferred, target did not revert |
| `false` | Call failed — target reverted, OOG, or invalid — ETH NOT transferred |

**Returndata** is NOT decoded — left in the returndata buffer; read with `last_returndata()` if needed. `call.call<M>()` decodes it for you; `raw_call` does not.

**Security: always check the bool**

```fe
// ❌ DANGEROUS — failure silently ignored
call.raw_call(addr: to, gas: 100000, value: balance, args_offset: 0, args_len: 0)
store.balance = 0   // balance zeroed even if ETH never left the contract!

// ✅ CORRECT
let ok = call.raw_call(addr: to, gas: 100000, value: balance, args_offset: 0, args_len: 0)
assert(ok)          // reverts entire tx if transfer failed
store.balance = 0
```

EOA calls always return `true` (no code to revert), which is why the Hardhat tests pass either way — but a contract recipient can reject the call and return `false`.

### Compiler errors hit — and how to fix them

> Source: hands-on compile attempts, 2026-04-30

```
error: cannot resolve `EVM` as a type   →  use Call not EVM (case matters)
error: undefined variable `evm`         →  use call.raw_call() not evm.raw_call()
error: `uses` typed parameters use      →  uses (evm: mut Evm) not (mut evm: mut Evm)
       `name: mut Type`, not `mut name: Type`
```

**Root cause:** `raw_call` is a default method on `pub trait Call` — it is inherited by any `call: mut Call` handle. `Evm` implements `Call` in the test VM only. Production contracts never need `evm: mut Evm`.

```fe
// ❌ Wrong — Evm is test-only
pub contract Vault uses (ctx: Ctx, log: mut Log, call: mut Call, evm: mut Evm) { ... }
Release { to } uses (mut evm) { evm.raw_call(...) }

// ✅ Correct — Call trait has raw_call by default
pub contract Vault uses (ctx: Ctx, log: mut Log, call: mut Call) { ... }
Release { to } uses (mut call) { call.raw_call(...) }
```

## Key Takeaways

1. `call.call<M>()` always requires a typed `message` — the `MsgVariant<Sol>` bound makes empty calldata physically impossible in the type system. *Source: `effects.fe:368`*
2. `call.raw_call(addr, gas, value, args_offset: 0, args_len: 0)` is the true bare-ETH-send (confirmed compiles + 7/7 Hardhat tests) — use `call: mut Call`, never `evm: mut Evm`, in production contracts. *Source: `effects.fe:349`*
3. `raw_call` returns `bool` — **always check it** with `assert(ok)`; silently discarding the bool means a failed transfer zeros your storage balance without the ETH leaving. *Source: `effects.fe:356–366`*

> "After `call.raw_call(...)`, always `assert(ok)` — a discarded `false` means lost funds with no revert."
