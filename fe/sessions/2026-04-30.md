## Theory

### `call.call<M>()` — definition

> Source: `effects.fe:368` — `pub trait Call`

```fe
fn call<M>(
    mut self,
    addr: own Address,
    gas: u256,
    value: u256,
    message: own M,        // ← ALWAYS required — cannot be omitted
) -> M::Return
where M: MsgVariant<Sol> + Encode<Sol>, M::Return: Decode<Sol> + AbiSize
```

**What it does internally** (`effects.fe:653–663`):

```
1. encode_calldata(M::SELECTOR, message)  → (args_offset, args_len) in scratch memory
2. raw_call(addr, gas, value, args_offset, args_len)
         ↓ if false → bubble_last_revert()   ← auto-bubbles callee revert
3. decode_returndata(last_returndata())    → M::Return
```

---

### Why empty calldata / skip `message` is impossible in `call.call()`

> Source: `effects.fe:368–369`, `fe-vs-solidity.md` (confirmed in practice)

| Solidity | Fe |
|---|---|
| `addr.call{value: x}("")` — empty calldata OK | `call.call(...)` — `message` required by type system |
| Triggers `receive()` on target | No `receive()` or `fallback()` in Fe |
| `""` = zero-length bytes | No `Unit`/`()` shorthand in `MsgVariant<Sol>` |

Trait bound `where M: MsgVariant<Sol>` means `M` must be a declared `msg` variant with `#[selector = sol("...")]`. There is no escape hatch.

```fe
call.call(addr: to, gas: 2300, value: amount)          // ❌ compiler error — message required
call.call(addr: to, gas: 2300, value: amount, message: "")  // ❌ not a MsgVariant
```

### `call.raw_call()` — the real bare-ETH-send ✅ NEW CONFIRMED

> Source: `effects.fe:349` (default impl on `pub trait Call`), confirmed usage in `recv_fallback.fe:34–37`

```fe
fn raw_call(
    mut self,
    addr: own Address,
    gas: u256,
    value: u256,
    args_offset: u256,   // memory pointer to calldata
    args_len: u256,      // 0 = empty calldata
) -> bool               // returns bool — does NOT auto-bubble revert
```

**Bare ETH send with empty calldata:**

```fe
let ok = call.raw_call(
    addr: to,
    gas: 100000,
    value: balance,
    args_offset: 0,   // ignored when args_len == 0
    args_len: 0,      // ← empty calldata = Solidity's addr.call{value: x}("")
)
// must check ok manually — no auto-revert
```

```
EVM CALL opcode
  ├── calldata = empty (len 0)
  ├── EOA target → no code → succeeds immediately
  └── ETH transfers ✅
```

**Key: `raw_call` has a default impl on the `Call` trait** — any `call: mut Call` effect handle in a production contract gets it automatically. No `evm: mut Evm` needed.

### Comparison table

| | `call.call<M>()` | `call.raw_call()` |
|---|---|---|
| `message` required | ✅ always | ❌ not needed |
| Empty calldata | ❌ impossible | ✅ `args_len: 0` |
| On failure | auto-bubbles revert | returns `false` — you must check |
| Return type | `M::Return` (ABI decoded) | `bool` |
| Available in contracts | `call: mut Call` | `call: mut Call` (same trait) |
| Available in tests | `evm.call(...)` | `evm.raw_call(...)` |

## Key Takeaways

1. `call.call<M>()` always requires a typed `message` — the `MsgVariant<Sol>` bound makes empty calldata physically impossible in the type system. *Source: `effects.fe:368`*
2. `call.raw_call(addr, gas, value, args_offset: 0, args_len: 0)` is the true bare-ETH-send equivalent to Solidity's `addr.call{value: x}("")` — returns `bool`, no auto-revert. *Source: `effects.fe:349`, `recv_fallback.fe:34–37`*
3. `raw_call` has a default impl on `pub trait Call` — any contract with `call: mut Call` can use it without `evm: mut Evm`. *Source: `effects.fe:344–366`*

> "Use `call.raw_call(args_len: 0)` for bare ETH sends — it's the only way to send empty calldata in Fe."
