## Theory

### Fe 26.1 — Key Changes
> Source: Fe 26.1.0 CHANGELOG + `fe-vs-solidity.md` (updated this session)

| Feature | v26.0 | v26.1 |
|---------|-------|-------|
| Typed errors | ❌ `assert()` only | ✅ `#[error]` + `revert_error()` |
| Error string revert | ❌ | ✅ `assert_msg(cond, "msg")` — emits `Error(string)` selector `0x08c379a0` |
| Overflow revert data | ❌ empty | ✅ `Panic(0x11)` — readable by Foundry/Hardhat |
| `assert(false)` | empty revert | ✅ `Panic(0x01)` |
| `Result::unwrap()` | no selector | ✅ selector-prefixed for `#[error]` types |
| Ctx APIs | partial | ✅ `origin`, `coinbase`, `chainid`, `basefee`, `selfbalance`, `blockhash`, `prevrandao`, `gaslimit` |
| `block.timestamp` | ❌ | ❌ still missing — use `ctx.block_number()` |
| Compile-time assert | ❌ | ✅ `static_assert(expr)` |
| Prelude | partial | ✅ `sol`, `Bytes`, `Decode`, `AbiDecoder` auto-imported |

**New error tools in 26.1:**
```fe
// 1. assert_msg — Solidity-compatible Error(string)
assert_msg(balance >= amount, "insufficient balance")

// 2. #[error] — custom typed error with auto 4-byte selector
#[error]
struct InsufficientBalance { have: u256, want: u256 }
revert_error(InsufficientBalance { have: balance, want: amount })

// 3. static_assert — compile-time check
static_assert(MAX_FEE_BPS <= 10000)
```

### Higher-Kinded Types (HKTs)
> Source: `fe/ingots/core/src/functional.fe`, `03-hkts.fe`

#### The Kind System

```
What you write               Kind            What it means
────────────────────────────────────────────────────────────
u256, bool, Option<u256>     *               complete type — use directly
Option, Vec                  * -> *          type constructor — needs 1 type
Result<AddError, _>          * -> *          error fixed, value slot open
Result                       * -> * -> *     needs 2 types
```

**Lego analogy:**
```
Mold (kind * -> *)   +   Clay (kind *)   =   Brick (kind *)

Option           +   u256            =   Option<u256>         ✅
Result<E, _>     +   u256            =   Result<E, u256>      ✅
Result           +   AddError        =   Result<AddError, _>  🏭 ← still a mold!
Result           +   AddError + u256 =   Result<AddError, u256> ✅
```

---

#### Functor — transform inside, keep the box

```
C<A>  .map(f: A → B)  →  C<B>

Some(5)  .map(x*2)  →  Some(10)    ← transforms value
None     .map(x*2)  →  None         ← skipped safely, no crash
Ok(5)    .map(x*2)  →  Ok(10)
Err(e)   .map(x*2)  →  Err(e)       ← short-circuits
```

```fe
pub trait Functor where Self: * -> * {
    fn map<T, U, F: Fn<T, U>>(self: own Self<T>, _ func: F) -> Self<U>
}
// Option and Result<E> both implement Functor
impl Functor for Option { ... }
impl<E> Functor for Result<E> { ... }
```

---

#### Applicative — apply a wrapped function to a wrapped value

```
C<F>  .ap(  C<A>  )  →  C<B>      where F: A → B

Some(double)  .ap(  Some(5)  )  →  Some(10)
Some(double)  .ap(  None     )  →  None       ← either None → None
None          .ap(  Some(5)  )  →  None
Ok(double)    .ap(  Ok(5)    )  →  Ok(10)
Err(e)        .ap(  Ok(5)    )  →  Err(e)     ← first error wins
Ok(double)    .ap(  Err(e)   )  →  Err(e)
```

```fe
pub trait Applicative: Functor where Self: * -> * {
    fn pure<T>(_ value: own T) -> Self<T>
    fn ap<T, U, F: Fn<T, U>>(self: own Self<F>, _ value: own Self<T>) -> Self<U>
}
```

---

#### `where C: * -> *` — the kind bound syntax

```fe
// ❌ BROKEN — compiler sees C as kind *, but C<u256> requires kind * -> *
fn add_in<C: Applicative>(a: own C<u256>, b: own C<u256>) -> C<u256> { ... }

// ✅ FIXED — where clause declares C is a type constructor
fn add_in<C: Applicative>(a: own C<u256>, b: own C<u256>) -> C<u256>
    where C: * -> *
{ ... }
```

**Rule:** Kind bounds go in `where` clauses, not in angle brackets.
Same pattern as stdlib: `pub trait Functor where Self: * -> *`

---

#### Smart contract use cases

| Use case | Pattern |
|----------|---------|
| Safe balance merge | `add_in(a: get_balance(user), b: get_staking_bal(user))` → `Option<u256>` |
| Fallible arithmetic | `add_in(a: Ok(rewards), b: Ok(bonus))` → `Result<E, u256>` |
| Generic fee application | `fn apply_fee<C: Functor>(amt: C<u256>) -> C<u256> where C: * -> *` |

## Key Takeaways

1. Kind `* -> *` = type constructor (mold, not brick) — declare with `where C: * -> *` in a where clause, same as Fe stdlib. *Source: `fe/ingots/core/src/functional.fe`*
2. `Functor.map` transforms the value inside a container safely; `Applicative.ap` applies a wrapped function to a wrapped value — together they write generic arithmetic once, usable for Option, Result, or any container. *Source: `fe/ingots/core/src/functional.fe`*
3. Fe 26.1 closes the Solidity error-handling gap: `#[error]`, `assert_msg`, `static_assert`, and `Panic(uint256)` revert payloads are now available. *Source: Fe 26.1.0 CHANGELOG*

> "A kind `* -> *` is a mold — declare it with `where C: * -> *` so the compiler knows it needs clay before it becomes usable."
