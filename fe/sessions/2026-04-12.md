## Theory Notes

> Source: `fe/ingots/std/src/evm/effects.fe` (confirmed from local Fe compiler source)

---

### Key Mental Model

Fe models execution capabilities as **traits**. The concrete type `Evm` implements them all.
When you write `ctx: Ctx` or `call: mut Call` in a `uses` clause, you're declaring which trait
capability the function needs. The compiler injects the `Evm` instance automatically.

```
Ctx      → read blockchain context (sender, value, block number…)
Call     → make external calls to other contracts
Log      → emit events
Create   → deploy new contracts (CREATE / CREATE2)
RawMem   → low-level memory ops (mload/mstore)
RawOps   → calldataload, keccak256, revert, etc.
Super    → all of the above combined (Ctx + RawOps + Log + Create + Call)
Evm      → concrete struct that implements every trait above
```

### `Ctx` trait — full API (14 methods)

Source: `effects.fe:146–161`

```fe
pub trait Ctx {
    fn address(self) -> Address      // address(this)
    fn caller(self) -> Address       // msg.sender
    fn origin(self) -> Address       // tx.origin
    fn coinbase(self) -> Address     // block.coinbase
    fn value(self) -> u256           // msg.value (wei)
    fn timestamp(self) -> u256       // block.timestamp
    fn block_number(self) -> u256    // block.number
    fn prevrandao(self) -> u256      // block.prevrandao
    fn gaslimit(self) -> u256        // block.gaslimit
    fn chainid(self) -> u256         // block.chainid
    fn basefee(self) -> u256         // block.basefee
    fn selfbalance(self) -> u256     // address(this).balance
    fn blockhash(self, _: u256) -> u256
    fn gas(self) -> u256             // gasleft()
}
```

Previous sessions only documented 4 (`address`, `caller`, `value`, `block_number`).
The full set is 14.

### `Call` trait — full API (4 methods)

Source: `effects.fe:344–376`

```fe
pub trait Call {
    // Regular call — can send ETH, target can change state
    fn call<M>(mut self, addr: own Address, gas: u256, value: u256, message: own M) -> M::Return
        where M: MsgVariant<Sol> + Encode<Sol>, M::Return: Decode<Sol> + AbiSize

    // Static call — read-only, no ETH, target cannot change state (STATICCALL opcode)
    fn static<M>(mut self, addr: own Address, gas: u256, message: own M) -> M::Return
        where M: MsgVariant<Sol> + Encode<Sol>, M::Return: Decode<Sol> + AbiSize

    // Delegate call — runs target's code in THIS contract's storage/context
    fn delegate<M>(mut self, addr: own Address, gas: u256, message: own M) -> M::Return
        where M: MsgVariant<Sol> + Encode<Sol>, M::Return: Decode<Sol> + AbiSize

    // Low-level escape hatch — returns bool, no auto-revert, no ABI decoding
    fn raw_call(mut self, addr: own Address, gas: u256, value: u256,
                args_offset: u256, args_len: u256) -> bool
}
```

All three high-level methods (`call`, `static`, `delegate`) require `mut Call` —
even `static`, because crossing a trust boundary is always a side effect in Fe.

### `Address::call` sugar

Source: `effects.fe:378–387`

```fe
impl Address {
    pub fn call<M>(self, _ message: own M) -> M::Return
        uses (call: mut Call)
    {
        call.call(addr: self, gas: ops::gas(), value: 0, message)
    }
}
```

Shorthand: forwards all available gas, sends 0 ETH, reverts on failure.

```fe
// Verbose:
call.call(addr: vault_addr, gas: 100000, value: 0, message: VaultMsg::Release { to })

// Sugar (same thing, all-gas, no ETH):
vault_addr.call(VaultMsg::Release { to })
```

Use `call.call(...)` directly when you need gas control or ETH value.

### What happens on call failure

Source: `effects.fe:652–691` (`impl Call for Evm`)

```fe
fn call<M>(...) {
    let out = encode_calldata(selector, message)   // 1. ABI-encode
    if !self.raw_call(addr, gas, value, ...) {
        bubble_last_revert()                        // 2. callee reverted → we revert too
    }
    decode_returndata(last_returndata())            // 3. success → ABI-decode and return
}
```

Fe **automatically bubbles reverts**. If the callee reverts, `call.call()` copies the
callee's revert data and reverts the current call with it. No silent failures.
`raw_call()` is the escape hatch if you want to handle failure yourself (returns `bool`).

### The two-layer declaration rule

```fe
// Layer 1: contract ceiling — required for ANY handler to use call
pub contract Arbiter uses (ctx: Ctx, call: mut Call) { ... }

// Layer 2: per-handler declaration — required at actual use site
Approve { vault, to } uses (mut store, ctx, mut call) {
    call.call(addr: vault, gas: 100000, value: 0, message: VaultMsg::Release { to })
}
```

- Missing Layer 1 → compiler error: `call` not in scope for the whole contract
- Missing Layer 2 → compiler error: `call` not in scope for this handler
- Handler cannot escalate beyond the contract ceiling

---

### The `msg` type as interface

In Fe, the callee's `msg` block IS the interface. Import it in the caller — no separate
interface file, no ABI drift. Compiler enforces alignment at the type level.

```fe
// Caller imports and uses the callee's msg type directly:
call.call(addr: vault, gas: 100000, value: 0, message: VaultMsg::Release { to })
//                                                      ^^^^^^^^
//                                  same type defined in Vault — one source of truth
```

---

### Security auditor lens

| Observation | Implication |
|---|---|
| `call: mut Call` in contract header | Contract CAN make external calls — check CEI order |
| `mut call` in handler `uses` | This handler makes external calls — audit this specifically |
| No `call` in header | Reentrancy via outgoing calls is impossible — compiler-enforced |
| `mut call` before `mut store` write | Potential reentrancy — storage not committed before external call |
| `call.delegate(...)` | Runs external code in this contract's context — high-risk, proxy pattern |
| `raw_call(...)` | Low-level, no auto-revert — check for unchecked return value |
