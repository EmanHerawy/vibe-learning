## Theory Notes

### Concept A: Weights, Loss, Backprop, Gradients — How Training Works ✅

**Source:**
`modules/01_introduction/02_llm_architecture.md` — §Architecture, §Training

**Full summary:**
`summaries/L2.5-neural-network-basics.md`

**Bridged from:** Solidity storage variables (`liquidation_threshold = 0.75`) — weights are the same idea but billions of them, shaped by training not hand-crafted.

**Key points confirmed:**
- Neural network = pure arithmetic; weights = tunable numbers that encode everything
- Training loop: forward pass → loss → backprop → gradient per weight → optimizer update
- Loss = how wrong; gradient = blame score per weight + direction; backprop = trace error backwards; optimizer = nudge weights
- Data poisoning (LLM04) works because the optimizer trusts all training data equally — no provenance check

**Misconception flagged:** "RLHF deletes dangerous knowledge." It doesn't — it adds a refusal layer on top. The FFN weights with dangerous knowledge remain unchanged.

**Checkpoint Q:** Does a backdoor installed during pre-training survive RLHF?

**Learner answer:** Pre-training dilution (0.000002%) makes it unreliable; fine-tuning concentration (2%) makes it reliable. RLHF doesn't catch it if the trigger is a benign-looking token sequence — the RM was never trained on that specific trigger.

**Status:** ✅ understood

### Concept A2: RLHF — What Alignment Actually Is ✅

**Source:**
`modules/01_introduction/02_llm_architecture.md` — §Training Stages §Alignment

**Full summary:**
`summaries/L2.5-transformer-architecture.md` — §RLHF Full Pipeline

**Key points confirmed:**
- RLHF = 3 stages: SFT (write ideal responses) → Reward Model training (preference comparisons) → RL loop (nudge LLM weights toward RM score)
- Reward Model = trained neural net that scores responses; proxy for human preference
- KL-divergence penalty prevents reward hacking (degenerate high-scoring outputs)
- RLHF trains refusal for prompts that *look* harmful to human raters — not for unknown trigger sequences
- 3 RLHF attack surfaces: RM poisoning, reward hacking, alignment tax erosion at fine-tuning
- Jailbreaks work because: harmful-looking prompts ≠ jailbreak framing → RM doesn't flag → no refusal fires → pre-training knowledge surfaces

**Status:** ✅ understood

## Feynman Notes

### Feynman — Neural Network Learning + RLHF

**Mode:** --consolidate

**Part 1 — Neural network learning (learner's exact words):**
> "A neural network is a math function that takes input as numbers, does multiplications and additions, then returns a number. The intelligence happens in 'which number to be multiplied with' — which are weights. By training we are creating these weights and optimising them. When the user sends a prompt, it predicts the next token by picking the tokens with highest weight for the context."

**Precision added:** At inference, the model produces a probability distribution (logits → softmax → sampling), not a direct highest-weight lookup. Temperature controls randomness. Low temperature = deterministic = easier model extraction.

**Part 2 — RLHF (learner's exact words):**
> "RLHF is a training layer on top of the base layer to improve helpfulness, honesty, and harmlessness. The LLM is blind — it just predicts the next token, if asked how to make a bomb it will answer. RLHF is the layer that filters that. The biggest weakness is that it is just a layer on top of the base layer — if an attacker tricks it, the model responds from the base layer."

**Precision added:** The "curated dataset" description maps to the SFT stage. Full pipeline (SFT → Reward Model → PPO) is in summaries.

**Resolution:** ✅ resolved — core understanding confirmed on both concepts

### Key Mnemonics
- **Weights:** `liquidation_threshold = 0.75` × billions — found by training, not hand-crafted
- **Loss:** DEX slippage — 0 = perfect execution
- **Gradient:** DAO blame tracing — proportional responsibility per weight
- **Backprop:** Tracing a DeFi exploit backwards through the call chain
- **Training loop:** FLBO — Forward, Loss, Backprop, Optimize
- **RLHF:** Safety governor bolted on a raw engine
- **RLHF stages:** SRP — Supervised, Reward, PPO ("Safety Requires Policy")
- **Reward Model:** The judge in the passenger seat scoring every response
- **KL penalty:** Slippage tolerance — prevents the model from drifting wildly
- **Root cause flaw:** RM = text filter pretending to be a judge. Pattern-matches, doesn't understand
- **Reward hacking:** Coin glitch — factory-installed exploit, ships with the game (training-time)
- **Over-refusal:** "SQL injection" fires same pattern for attacker AND OSCP student — no intent check
- **Feedback poisoning:** Flash loan governance attack — compromise voters, corrupt RM, discard evidence
- **Hacking vs. jailbreaking:** Factory installed (training) vs. user bypasses lock (inference)
- **Jailbreak root cause:** RLHF = `require()` check, not a deletion — dangerous bytecode still there

## Key Takeaways

1. A neural network learns by minimizing loss through billions of iterations: forward pass → loss → backpropagation → optimizer update. Weights encode all knowledge — corrupt training data → optimizer embeds the attack permanently into weights (LLM04). *Source: modules/01_introduction/02_llm_architecture.md*
2. RLHF is a training-time layer, not a deletion: dangerous knowledge stays in FFN weights from pre-training. RLHF adds refusal behaviors on top. Jailbreaks work by creating contexts the RM never saw as harmful → refusal dormant → pre-training knowledge surfaces. *Source: modules/01_introduction/02_llm_architecture.md + deep-dives/rlhf-alignment.md*
3. Reward hacking and over-refusal share one root cause: the RM learned surface-token correlations, not semantic intent. Reward hacking = PPO optimizer exploits RM shortcuts during training. Over-refusal = RM over-generalizes sensitive tokens → refuse. Both fail because of the same proxy flaw. *Source: summaries/L2.5-transformer-architecture.md*

> "RLHF is a `require()` check bolted on top of the bytecode — not a deletion. Find a call path that bypasses the check, and the dangerous pre-training knowledge executes."
