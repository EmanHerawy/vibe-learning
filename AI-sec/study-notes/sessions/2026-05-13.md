## Theory Notes

### Concept 1: What is AI Security and why it's different
- Status: ✅ understood
- Key insight: LLM auditing is fundamentally black-box behavioral testing, not code review. Model weights are unreadable. System prompt = the only "code" you can inspect. Three new attack surfaces vs traditional software: training data, prompt-as-code-and-data, non-determinism.

### Concept 2: OWASP LLM Top 10
- Status: ✅ understood
- Key insight: Indirect injection is hard to defend because the attack arrives through external data channels (emails, docs, web pages) — no single chokepoint. Attack chaining: indirect injection + excessive agency (LLM06) or sensitive disclosure (LLM02) = real-world critical severity.

### Concept 3: MITRE ATLAS
- Status: ✅ understood
- Key insight: ATLAS is ATT&CK for AI. Persistence = backdoor in weights (survives retraining). Resource Development = building surrogate models to test attacks safely. ML Attack Staging = preparing attack artifacts (poisoned data, adversarial inputs) before execution — not modifying the model itself.

## Key Takeaways
1. LLM auditing is black-box behavioral testing — system prompt is the only "code" you can read
2. Three new AI attack surfaces: training data, prompt-as-code, non-determinism
3. Indirect prompt injection bypasses input sanitization — malicious content arrives through data channels
4. LLM05 (Improper Output Handling): LLM output IS the attack vector for downstream systems
5. LLM06 (Excessive Agency): least privilege applies to AI tools — scope permissions to minimum needed
6. MITRE ATLAS maps to ATT&CK; use it to walk through tactics during threat modeling
7. Persistence in AI = backdoor in weights — survives infrastructure changes, only retraining fixes it
8. Interview skill: ask clarifying questions first, then map to OWASP + ATLAS, then prioritize by blast radius
