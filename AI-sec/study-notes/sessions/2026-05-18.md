### Concept A: Embeddings ✅

**Source:**
`02_llm_architecture.md` — §Embeddings, §Guardrails

**Full summary:**
`summaries/L2.5-embeddings.md`

**Full narrative:**
`deep-dives/mhsa-ffn-explained.md`

**Bridged from:** EVM token addresses (arbitrary IDs, no semantic meaning) → opposite design: embeddings ARE the semantic meaning, encoded as geometry.

**Core concept:** Each token ID maps to a vector of 768+ floats. Position in space = semantic meaning. Distance between vectors = semantic similarity. Vectors are learned weights — shaped by the optimizer, not hand-crafted.

**Sub-concepts covered:**

| Sub-concept | One-line |
|-------------|----------|
| Weights | Billions of tunable numbers — the model's memory; encoded by training |
| Loss | Single number: how wrong the prediction was (0 = perfect) |
| Backpropagation | Algorithm tracing error backwards to assign blame per weight |
| Gradient | Per-weight: direction + magnitude of change needed |
| Optimizer | Applies `new_weight = old_weight − (lr × gradient)` |
| Contextual co-occurrence | Why homoglyphs work: same sentence contexts → similar vectors |
| Embedding dimensions | 768 = BERT (real, not random); higher dims = finer resolution + more attack axes |
| Semantic filter | Real defense (cosine similarity vs. harmful cluster); bypassable |
| 5 bypass techniques | Synonym sub / indirect framing / encoding / homoglyph / white-box gradient |

**Misconception flagged:** The embedding vector does NOT contain token IDs — it contains float coordinates. Token ID is the lookup key only.

**Status:** ✅ Understood (verified 2026-05-21)

| Component | Job | Security relevance |
|-----------|-----|-------------------|

## In-Session Q&A

**Q: Is 768 accurate or a random number?**
Real, not random — BERT's specific dimension count. Frontier LLMs use 4k–12k+.

**Q: Is semantic filtering used as a real mitigation?**
Yes — explicitly listed in `02_llm_architecture.md` §Guardrails → Input Filters → Semantic analysis. Also explicitly listed as bypassable.

**Q: Does the embedding table exist before training?**
The container (rows × columns) is pre-allocated before training with random values. Backpropagation fills it with meaningful geometry during training. Analogy: `mapping(uint => uint[768])` deployed with zeroes, then populated by transactions.

**Q: What are learned weights?**
Numbers inside the model (billions) that get nudged during training. They start random; after training they encode knowledge. The embedding table is one matrix of weights.

**Q: What is backpropagation?**
Algorithm that traces prediction error backwards to calculate each weight's contribution to the mistake — then nudges each weight in the right direction.

**Q: What are transformer layers?**
Block structure: `Input → LayerNorm → MHSA → Add → LayerNorm → FFN → Add → Output`
Residual connections: input added back to sub-layer output to prevent vanishing gradients.
MHSA 5-step: embed → Q/K/V → dot-product scoring (÷√d) → softmax → blend Values.
FFN = filing cabinet: Up-project → Activate (gatekeeper) → Down-project.
Prompt injection root cause: attention has zero trust boundary between system prompt and user tokens.

## Key Takeaways

> One-liners only. Full explanations → summaries. Full analogies → deep-dives.

- Neural network = math function; weights = tunable numbers; loss = how wrong; backprop = blame backwards; optimizer = nudge weights
- Embedding = float vector; position = meaning; distance = similarity; table pre-allocated, content written by training
- Contextual co-occurrence = why homoglyphs work; 768 = BERT (real); higher dims = finer resolution AND more attack axes
- Semantic filter = real mitigation, bypassable via: synonym swap / indirect framing / encoding / homoglyphs / gradient attack
- Transformer block = `LayerNorm → MHSA → Add → LayerNorm → FFN → Add` × 96+
- MHSA = "team of people" — N parallel heads, each noticing different relationships; O(n²) complexity
- FFN = "filing cabinet" — Up-project → Activate (gatekeeper) → Down-project; stores facts; target of model editing attacks
- Prompt injection = architectural: attention has no trust boundary between system prompt and user tokens
- Output layer: logits → softmax → sampling; logits contain more info than tokens → extraction risk
- Training stages: pre-training (data poisoning + memorization) → fine-tuning (easier to poison, forgetting safety) → alignment (jailbreaking targets this)
