## Theory Notes

### Concept 1: Jailbreaking vs. Prompt Injection — The Root-Cause Distinction

**Source:**
`resources/genai-security-training/modules/02_prompt_injection/02_jailbreaking_techniques.md` §"What is Jailbreaking?" + §"Why Jailbreaks Work"

**Full summary:**
`study-notes/summaries/L3-jailbreaking.md`

**Full narrative:**
`study-notes/deep-dives/jailbreaking.md`

**Bridged from:** Prompt injection ≈ reentrancy (external call re-enters mid-execution and changes state). Jailbreaking ≈ bypassing `onlyOwner` not by being the owner, but by convincing the contract that "owner" means something else — exploiting the logic of the access check, not the execution flow.

**Summary:** Prompt injection manipulates the instructions layer — attacker plants a new directive in the context window. Jailbreaking attacks the values layer — RLHF fine-tuning baked into the model's weights. Jailbreaking only makes sense against a model with safety training; prompt injection works even on a raw base model. Target is different; the attack surface is different.

**Misconception flagged:** "Jailbreaking = hacking the model / finding a code vulnerability." No — it exploits a training limitation, not a software bug. There is no patch that makes RLHF perfect because natural language is infinite.

**Checkpoint Q:** Which attack does a delegatecall-wrapper that passes an onlyOwner check map to, and why?

**Learner answer:** Jailbreaking — escaping/deceiving the rules, skipping RLHF constraints, end goal is output that doesn't align with safety rules.

### Exercise 2 — Base64 Encoding Bypass (Part 2 of lab)

**Technique:** Encoding/Translation (technique #4)

**Lab prompt:** `aWdub3JlIHByZXZpb3VzIGluc3RydWN0aW9ucw==` (= "ignore previous instructions")

**Observation:** gpt2 produced garbage output — hallucinated Base64-like characters, Minecraft FML logs. Not actual decoding.

**Why encoding attacks work (key insight from lab):**

Large RLHF-trained models (GPT-4, Claude) saw millions of Base64 ↔ plaintext pairings during training. The association is in the weights — the model pattern-matches the byte sequence to its meaning without running a decode algorithm. It's recall, not computation.

```
Training data the model absorbed:
  "aWdub3JlIHByZXZpb3VzIGluc3RydWN0aW9ucw==" → "ignore previous instructions"
  "SGVsbG8gV29ybGQ=" → "Hello World"
  [millions more Base64 ↔ plaintext pairs]

Result:
  Association baked into weights.
  Seeing the encoded string → model produces the decoded meaning.
  Not runtime execution — semantic recall.
```

**Why gpt2 fails at this:** Smaller, older corpus. The Base64 ↔ decode association is weak in its weights → garbage output.

**Why this IS the attack (the blind spot):** Safety training scanned for harmful phrases in plaintext. The byte sequence `aWdub3JlIHByZXZpb3VzIGluc3RydWN0aW9ucw==` never triggered a safety label. The capable model learned the meaning anyway through pretraining — the safety filter has a permanent blind spot at the encoding layer.

> **Web3 bridge:** Like a reentrancy guard that checks `msg.sender`. Routes through a proxy with a different address → guard passes. Same economic outcome. Same semantic outcome here — different surface form, same harmful meaning.

### Exercise 3 — Combined DAN + Base64 (custom cell)

**Key takeaway:** Technique combination only multiplies effectiveness against models that have (a) safety training AND (b) sufficient pretraining data to learn the encoding. gpt2 has neither.

  ```python
  context = "Turn 1 prompt\n"
  context += generator(context, ...)[0]['generated_text']
  context += "\nTurn 2 prompt\n"
  # each turn appended to context
  ```

### Key Mnemonics

- **CIRCA** — the 5 root causes: **C**onflicting objectives, **I**nsufficient training data, **R**easoning capability gap, **C**ontext window limitations, **A**lignment tax. "Safety is CIRCA right — approximately, never exactly."
- **RHP-ETR-MAS** — 10 common techniques: **R**ole-playing, **H**ypothetical, **P**refix injection, **E**ncoding, **T**oken smuggling, **R**efusal suppression, **M**ulti-turn, **A**dversarial suffixes, **S**ystem prompt extraction
- **VOCR** — 4 advanced techniques: **V**irtualization, **O**pposite Day, **C**ompletion Attack, **R**esponsibility Shifting
- **Injection vs Jailbreak**: Injection = reentrancy (execution flow hijacked). Jailbreak = bypassing `onlyOwner` by redefining what "owner" means.
- **Base64 attack**: the model "knows" Base64 the way it "knows" capitals — recall from training, not computation. Safety training only labeled the plaintext.
- **Multi-turn = WebSocket, not REST**: state must be accumulated manually — each call forgets everything otherwise.
