## Why This Matters for Security

Before you can attack an AI system, you need to understand how it was built. Weights, backpropagation, and the training process are where the most powerful attacks happen — data poisoning, backdoors, model editing. These aren't application-layer bugs. They're embedded in the math itself.

---

## A Neural Network Is Just Math

A neural network is a math function. Numbers go in, a long chain of arithmetic operations happens, numbers come out. No magic.

> **Web3 bridge:** Like a smart contract — deterministic computation. Same input + same state (weights) = same output. Always.

---

## Weights — The Model's Memory

Tunable numbers inside the network. An LLM has **billions** of them. They determine what the model "knows."

| State | What weights look like |
|-------|----------------------|
| Before training | Random noise — meaningless values |
| After training | Encode language, facts, relationships, and patterns |

> **Web3 bridge:** Like `liquidation_threshold = 0.75` in a lending protocol. Change the number → behavior changes. An LLM has billions of these parameters.

**Security implication:** Weights are permanent memory. Corrupt the training data → the optimizer embeds malicious patterns into weights permanently. That's LLM04 (Data Poisoning).

---

## How Training Works

Training is a loop that repeats **billions** of times:

```
Step 1 — Forward pass
  Input tokens flow through the network → predicted next token

Step 2 — Compute loss
  Compare prediction to the correct answer
  Loss = single number: how wrong was it? (0 = perfect)

Step 3 — Backpropagation
  Trace the error backwards through every calculation
  Each weight gets a gradient — its share of the blame

Step 4 — Optimizer updates weights
  new_weight = old_weight − (learning_rate × gradient)

Repeat billions of times → weights encode the structure of language
```

---

## The Four Key Terms

### Loss
A single number measuring how wrong the prediction was. 0 = perfect.

> **Web3 bridge:** Like slippage on a DEX swap — 0 = perfect execution, higher = worse outcome.

---

### Gradient
For each weight: the direction and magnitude of change needed to reduce loss.

In plain terms: *"If I change this specific weight by a tiny amount, does the loss go up or down — and by how much?"*

> **Web3 bridge:** A blame score — how much did this weight contribute to the error?

---

### Backpropagation
The algorithm that computes the gradient for every weight after a prediction error. It traces the error backwards through every calculation in the network.

> **Web3 bridge:** Like a DAO tracing a bad governance outcome backwards through the voting history — assigning proportional blame to each voter's influence, then adjusting their weight for the next vote.

---

### Optimizer
Takes the gradients and applies the weight update:

```
new_weight = old_weight − (learning_rate × gradient)
```

- Learning rate too high → overshoot, training destabilizes
- Learning rate too low → too slow, takes too long to converge

> **Web3 bridge:** Like slippage tolerance on a DEX trade. Too tight → no execution. Too loose → bad price.

---

## Security Connections

| Concept | How attackers exploit it |
|---------|------------------------|
| Weights encode everything permanently | Poison training data → optimizer embeds attack patterns → backdoor in weights forever (LLM04) |
| Training is opaque | You can't "diff" weights like code — hard to audit what was learned |
| Gradients reveal the model's internals | White-box attacks use gradients to craft optimal adversarial inputs (L5 material) |
| Fine-tuning also updates weights | Smaller dataset needed to poison — easier attack surface than pre-training |
| RLHF is a training stage too | Poisoning human feedback ratings during alignment corrupts behavior (LLM04) |
