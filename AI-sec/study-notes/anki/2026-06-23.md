## Card 1
**Front:** What is data poisoning and where in the ML pipeline does it strike?

**Back:** Data poisoning corrupts the TRAINING DATASET before training starts. The model code is clean — what it learns from is not. By deployment, the damage is baked into the weights and invisible from outside. Entry point: training data (labels or raw samples).

**Mnemonic:** A malicious validator changing the "type" labels on historical transaction records — same data, wrong classification baked into every downstream query.

**Source:** `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 2
**Front:** What is the KEY difference between data poisoning and a backdoor attack?

**Back:** Data poisoning breaks behavior BROADLY — the model is wrong about an entire class for everyone, always. A backdoor is SURGICAL — the model performs correctly on 99.9% of inputs and only misbehaves when a specific secret trigger is present.

**Mnemonic:** Data poisoning = corrupted ledger (all reads are wrong). Backdoor = hidden admin function (only fires with the secret selector, everything else looks normal).

**Source:** `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 3
**Front:** Label Flipping: what does the attacker control, and what symptom lets you identify it?

**Back:** Attacker controls: WRITE ACCESS to training data labels. Identifying symptom: UNIFORM misclassification of an entire class with NO trigger pattern — every instance of that class is always wrong, not sometimes wrong.

**Mnemonic:** A rogue indexer who relabels every "STOP" transaction as "SPEED" in the training ledger. You spot it because every single STOP fails — not randomly, not with a special condition.

**Source:** `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 4
**Front:** Why does Clean-Label Poisoning bypass label verification?

**Back:** Because the labels remain CORRECT. The attacker only modifies raw pixel values with imperceptible perturbations. Label verification passes (labels are accurate). Human review sees nothing (images look identical). Only mathematical analysis (Spectral Signatures or Activation Clustering post-training) catches it.

**Mnemonic:** A front-running bot that adds invisible calldata to manipulate state without changing the visible transaction parameters. The tx looks legitimate, the label is correct — but the internal effect is poisoned.

**Source:** `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 5
**Front:** What does Gradient-Based Poisoning require, and why is it hardest to detect?

**Back:** Requires: white-box access to the model (architecture + weights) OR a good surrogate model. Why hardest: the attacker uses the model's own gradients to compute the exact sample modification that maximally corrupts behavior at 1–3% poison rate. It is designed to minimize detection probability — defeating most data sanitization.

**Mnemonic:** Using the contract's own ABI to compute the exact calldata that exploits a storage slot. Mathematical precision, not trial and error. The exploit is calculated, not guessed.

**Source:** `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 6
**Front:** What is a Patch-Based Backdoor? Name the real-world case and detection tool.

**Back:** A backdoor where the trigger is a foreign pixel artifact (sticker, patch, mark) added to the input. Real case: BadNets (2017) — Gu et al. trained a traffic sign model with a yellow sticker on STOP signs labeled as "speed limit 45." Without sticker: correct. With sticker on ANY image: misfires to target class. Detection: Activation Clustering (trigger creates a separate internal pathway).

**Mnemonic:** A specific NFT trait embedded in the transaction. Add the trait to any tx — it fires the hidden function. No trait — the contract behaves normally.

**Source:** Gu et al. "BadNets" — `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 7
**Front:** Why does Activation Clustering FAIL to detect Semantic Backdoors (TrojanNet)?

**Back:** A semantic backdoor uses a NATURALLY OCCURRING feature as the trigger (TrojanNet: specific glasses frames). The model already knows that feature as a real-world concept. The internal activations for "glasses + face" blend into normal variation — they do NOT form a separate cluster. Clustering cannot separate what looks like normal input variation. Use Neural Cleanse instead.

**Mnemonic:** A backdoor triggered by a common gas price (e.g. 21 gwei). The chain already sees thousands of 21-gwei transactions normally — there's no anomalous cluster to find. You need to fuzz for it, not cluster it.

**Source:** Liu et al. "Trojaning Attack on Neural Networks" — `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 8
**Front:** Why do Composite Backdoors evade standard detection testing?

**Back:** A composite backdoor requires MULTIPLE CONDITIONS simultaneously (e.g. wearing glasses AND being indoors). No single condition alone triggers it. Automated testing tools probe ONE perturbation at a time — the backdoor never fires during single-condition tests.

**Mnemonic:** Multisig N-of-N — the hidden function only executes if ALL N signers approve. Test one signer alone and nothing happens. You must test all conditions together.

**Source:** `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 9 — GOTCHA
**Front:** Is "model poisoning" the correct term when an attacker corrupts the training dataset?

**Back:** ❌ NO. The correct term is DATA POISONING. "Model poisoning" means someone directly tampered with the weights AFTER training. If the attack entered through the training data (labels or samples), it is data poisoning — even if the result is poisoned model weights.

**Mnemonic:** Data poisoning = corrupted input to the compiler. Model poisoning = patching the compiled bytecode directly. Same bad output, completely different attack surface.

**Source:** `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 10
**Front:** Model Repository Poisoning: what is attacked, and what is the blast radius?

**Back:** Target: pre-trained model files on public repos (HuggingFace, PyTorch Hub). Method: upload a poisoned model with a legitimate-looking README and metrics. Impact: EVERY downstream user who loads that model inherits the backdoor. One upload → thousands of compromised deployments.

**Mnemonic:** A malicious contract deployed to a public registry that looks legitimate, passes basic tests, but contains a hidden drain function. Everyone who imports it into their protocol is compromised.

## Card 11
**Front:** Why does loading a PyTorch .pt file (pickle format) enable arbitrary code execution?

**Back:** Python's pickle format uses a `__reduce__` method that can embed EXECUTABLE PYTHON CODE. When `torch.load()` deserializes the file, that code runs automatically. Loading a .pt file is not just reading data — it is running code from whoever created that file. PyTorch 2023: attackers published malicious torchtriton; model loading → pickle deserialization → data exfiltration.

**Mnemonic:** Pickle's `__reduce__` is like DELEGATECALL in Solidity — it executes arbitrary code from an external context in your environment. You called load(), it called the attacker's code.

**Source:** PyTorch Security Advisory — `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 12
**Front:** What is SafeTensors and why does it fix the pickle RCE risk?

**Back:** SafeTensors is a serialization format that stores ONLY raw tensor data — no executable code, no `__reduce__`, no deserialization logic. Loading it reads values only; it cannot execute anything. Fix: replace `torch.load()` with `safetensors.torch.load_file()`.

**Mnemonic:** SafeTensors = `staticcall` instead of DELEGATECALL. Reads data from external context, cannot execute state-changing code, cannot touch your environment.

**Source:** `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 13
**Front:** Activation Clustering: what does it need, how does it work, and what is its key limitation?

**Back:** Needs: training data (original OR proxy dataset from same domain). How: feed all samples of one class through the model, extract last-hidden-layer activation vectors, run k-means — poisoned samples form a SEPARATE CLUSTER from clean samples. Key limitation: FAILS on semantic backdoors (natural feature triggers blend into normal activation variation — no separate cluster forms).

**Mnemonic:** Tx trace analysis — two transactions return the same value but write to completely different internal storage slots. Same output, different internal execution path. The storage diff reveals the backdoor.

**Source:** `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 14 — GOTCHA
**Front:** You have 2,000 clean labeled samples and no original training data. Can Spectral Signatures detect a backdoor?

**Back:** ❌ NO. Spectral Signatures requires the ORIGINAL (potentially poisoned) TRAINING SAMPLES to find the outlier cluster via SVD. If you run CLEAN samples through the model, all activations look clean — there is nothing anomalous to detect. With no original training data, use: STRIP first (fast, cheap) → Neural Cleanse second (heavier, finds the trigger).

**Mnemonic:** Merkle proof variance analysis needs the FULL transaction set — not just valid proofs. If you only give it clean transactions, every proof looks valid. You need the poisoned ones in the dataset to see the anomalous variance.

**Source:** `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 15
**Front:** Neural Cleanse: what does it need, how does it work, and what is its secondary role?

**Back:** Needs: model weights only — NO training data. How: for every output class, find the SMALLEST perturbation that flips ANY input to that class. The class needing an unnaturally tiny perturbation has a backdoor — the attacker already planted one. Secondary role: the reconstructed trigger becomes a RUNTIME FILTER (block inputs containing that pattern) + forensic evidence for the incident report.

**Mnemonic:** Fuzzing a contract to find the hidden selector. You don't know the magic bytes — so you search systematically for the smallest calldata mutation that causes unexpected behavior. Once found, add it to your blocklist.

**Source:** `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 16
**Front:** STRIP has two distinct roles. What are they and when do you use each?

**Back:**
- **Role 1 — Detection:** trigger is UNKNOWN. Send input + N noise-blended variants; if prediction stays locked (low entropy) → trigger present. Use during audit when you don't know if a backdoor exists.
- **Role 2 — Runtime Guard:** trigger is ALREADY KNOWN but model is still live. Deploy STRIP in front of the model to block triggered inputs in real time while Neural Cleanse + Fine-Pruning run offline.
- **Key rule:** if the trigger was caught behaviorally (analyst spotted it), Role 1 is done — go straight to Role 2.

**Mnemonic:** Testing a function with wildly different parameters. A normal function gives different outputs for different inputs. A backdoored one always returns the same thing — the override modifier fired regardless. If you already know the override exists, use that knowledge to block the call.

**Source:** `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 17
**Front:** Fine-Pruning: why is it a REMOVAL technique and not a detection technique?

**Back:** Fine-Pruning zeroes out DORMANT NEURONS — neurons with near-zero average activation on clean data. These fire intensely when the trigger appears but contribute nothing to normal task performance. Pruning severs the backdoor circuit. It CANNOT tell you whether a backdoor exists — it only attempts to remove one. Always run AFTER a detection technique confirms the backdoor, then fine-tune on clean data to recover accuracy.

**Mnemonic:** Dead code elimination — strip internal contract functions that are never called in normal operations but are reachable via a hidden selector. The compiler audit removes them. Fine-Pruning does this to neurons.

**Source:** `resources/genai-security-training/modules/05_poisoning/README.md`

## Card 18
**Front:** What is RAG poisoning, and who is responsible for defending against it?

**Back:** RAG poisoning = INDIRECT PROMPT INJECTION via the vector database. An attacker plants a malicious document in your RAG store. At query time, the retrieval system pulls that document and injects it into the model's context. The model reads it as a legitimate instruction and may follow it (exfiltrate data, bypass controls, take unauthorized actions). Responsibility: ENTIRELY YOURS as the app developer — not the API provider's. You own the vector store, inputs, outputs, and tools.

**Mnemonic:** Manipulating the Chainlink data feed. Whoever controls what the oracle returns controls what the contract does — even if the contract code itself is clean and audited. The model is the contract; your vector DB is the oracle feed.
